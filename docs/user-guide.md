# Paper Agent 使用指南

Paper Agent 是一个自动追踪 **AI Infrastructure** 方向 arXiv 论文的工具，会用 AI 帮你筛选、分类、打分，并通过邮件把高质量论文推送到你的邮箱。你也可以打开 Web 页面随时浏览历史论文。

这份文档面向使用者，告诉你「能用它做什么」和「怎么快速用起来」。

---

## 你能得到什么

### 📬 每日论文推送邮件

订阅之后，每天会收到一封日报邮件，里面包含：

- 当天新发现的、AI 帮你筛选过的论文
- 每篇论文的 **影响力等级**（重磅突破 / 稳健工作 / 渐进改进）
- 每篇论文的 **中文摘要**、**关键贡献**、**问题陈述**、**方法概述**
- 论文所属的 **子领域标签**（如 serving、kv_cache、quantization 等）
- arXiv 原文链接

### 🌐 Web 浏览页面

打开 Web 页面可以：

- 浏览历史所有评分过的论文
- 按 **子领域**、**影响力等级**、**发表时间**、**关键词** 过滤
- 把筛选条件保存到浏览器，下次打开自动应用
- 在 `/subscribe` 自助订阅，无需找管理员

## 快速上手

### 一、订阅论文推送（最常用）

1. 打开浏览器访问 Paper Agent 的 Web 页面（例如 `http://your-server:8000`）
2. 点击页面上的 **「订阅」** 入口（或直接访问 `/subscribe`）
3. 填写：
   - 你的 **邮箱地址**
   - 你关注的 **子领域**（可以全选，也可以挑几个）
4. 提交后，订阅生效。每天定时（一般 9:00）会收到推送邮件

> 想退订或修改？联系管理员即可。

### 二、浏览历史论文

打开 Web 主页就能看到论文列表，默认展示 **稳健工作及以上** 的论文（隐藏渐进改进档）。

页面上可用的筛选方式：

| 控件 | 作用 |
|---|---|
| **顶部子领域 chip** | 点击某个标签只看该子领域 |
| **时间范围下拉** | 近一周 / 近一月 / 近 3 月 / 近半年 / 近一年 / 近 3 年 |
| **搜索框** | 按标题关键词搜索 |
| **偏好面板** | 切换「全部论文 / 自定义领域」、设最低影响力档位 |
| **分页** | 每页 25 篇，按影响力优先排序 |

筛选条件会被编码到 URL，可以收藏或分享给同事。

### 三、读懂论文卡片

每篇论文显示一个卡片，里面有几块视觉信息：

- **影响力徽章**
  - 🟠 **重磅突破**（橙色边框 + 琥珀色徽章）—— 可能改变实践的新技术，强烈推荐看
  - ⚪ **稳健工作**（灰色徽章）—— 工作扎实的有用改进
  - 🌫️ **渐进改进**（半透明卡片）—— 小幅变化、范围狭窄，默认隐藏
- **🟢 关键贡献**（绿色块）—— 1–3 条精炼的核心贡献
- **🔵 问题陈述**（蓝色块）—— 这篇论文要解决什么问题
- **🟣 方法概述**（紫色块）—— 用了什么方法
- **子领域标签** —— 论文属于哪个 AI Infra 子方向
- **评分** —— 相关度 + 质量综合分

排序规则：**先按影响力等级排，同等级内按综合得分排**。所以一篇「重磅突破」永远排在「稳健工作」之上。

---

## 影响力等级说明

| 等级 | 含义 | 默认显示 |
|---|---|---|
| **重磅突破** (`breakthrough`) | 可能广泛被引用或改变实践的新技术、新结果 | ✅ |
| **稳健工作** (`solid`) | 工作扎实，稳健推进基线的有用改进（大多数论文） | ✅ |
| **渐进改进** (`incremental`) | 小幅变化，范围狭窄，评估有限 | ❌ 隐藏 |

如果你只想看最值得读的，把偏好面板的「最低影响力」切到 **重磅突破**；想看全部，切到 **包含渐进改进**。

## 子领域分类

系统自动把论文分到以下 14 个 AI Infra 子领域：

`quantization`（量化）· `distillation`（蒸馏）· `pruning`（剪枝）· `sparsity`（稀疏）· `distributed_training`（分布式训练）· `parallelism`（并行）· `serving`（推理服务）· `speculative_decoding`（投机解码）· `kv_cache`（KV 缓存）· `moe`（MoE 专家混合）· `compiler`（编译器）· `memory_optimization`（显存优化）· `communication`（通信）· `scheduling`（调度）

订阅时按需勾选，邮件只会推送你订阅子领域的论文。

## 常见问题

**Q：订阅之后多久能收到第一封邮件？**
A：取决于服务的推送时间（一般每天 9:00 一次），如果当天没有新论文匹配你的订阅，可能会跳过。

**Q：能订阅多个邮箱吗？**
A：可以，分别在 `/subscribe` 用不同邮箱提交即可，每个邮箱独立。

**Q：邮件没收到怎么办？**
A：先检查垃圾邮件箱；如果确认没收到，联系管理员排查 SMTP 配置。

**Q：Web 上的筛选条件能保存吗？**
A：可以，偏好面板里的设置会自动保存到当前浏览器，下次访问会自动应用。换浏览器或清缓存后需要重新设置。

**Q：影响力等级是怎么判断的？**
A：由 AI 评分时直接给出，不是按分数阈值机械划分。所以即使分数相同，AI 也可能根据论文的新颖性、影响范围把它放进不同等级。

**Q：邮件里的「重要老作」分区是什么？**
A：在管理员启用引用感知功能后，系统会主动从 Semantic Scholar 发现高引用老论文（默认 2 年以前、≥100 次引用），打分入库后作为独立分区附在每日邮件末尾。这部分论文不占用 `top_n` 名额，是额外赠送，且每篇只会发送一次。

**Q：为什么有些论文标着「📈 N citations」，还有「🔖 重要老作」徽章？**
A：📈 徽章显示的是 Semantic Scholar 上的引用数。🔖 徽章表示这篇论文是反向发现的高引用老论文，而不是 arXiv 新提交。引用数本身不直接进入 total_score，但会让 AI 在影响力大涨的论文上做动态重判。

## 想要更多控制？

所有推送用户均通过 Web 表单订阅，统一使用全局邮件配置和推送阈值。如需调整推送阈值，请联系管理员修改 `config.yaml` 中的 `thresholds` 段——管理员可参考项目根目录的 `README.md` 和 `CLAUDE.md`。
