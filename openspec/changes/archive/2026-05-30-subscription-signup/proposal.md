## Why

当前 Web UI 设计较为朴素，用户体验有待提升。同时，系统缺少用户自助订阅机制——新用户只能通过手动编辑 `config.yaml` 添加，这对非技术用户不友好。通过增加自助订阅页面和优化 UI 设计，可以降低用户接入门槛，提升整体使用体验。

## What Changes

- **新增订阅注册页面**：用户可通过 Web 表单输入邮箱地址和感兴趣的子领域，系统自动将其添加到配置中并启用定期推送
- **UI 视觉升级**：重新设计现有页面样式，采用更现代的配色方案、更好的排版和交互反馈
- **订阅管理入口**：在主页添加导航链接，方便用户访问订阅页面
- **配置持久化**：将用户订阅信息保存到数据库或配置文件，确保重启后不丢失

## Capabilities

### New Capabilities
- `subscription-form`: 用户自助订阅表单，包括邮箱输入、子领域多选、提交验证
- `subscription-storage`: 订阅信息持久化存储机制
- `ui-redesign`: Web UI 整体视觉升级，包括配色、排版、响应式布局

### Modified Capabilities
None

## Impact

- **Web 前端**：新增订阅页面和表单组件，现有页面样式重构
- **后端路由**：添加订阅 API 端点处理表单提交
- **配置管理**：动态更新用户配置或数据库，支持运行时添加新用户
- **通知系统**：新订阅用户自动接入现有的 email 通知流程
- **数据库**：可能需要扩展 schema 以存储订阅来源等元数据
